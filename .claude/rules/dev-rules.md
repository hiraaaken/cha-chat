# 開発フロー

## Issue ベース開発

すべての作業は GitHub Issue を起点とする。

### ブランチ命名規則

```
feature_<Issue番号>
```

例: `feature_48`

### ワークフロー

1. GitHub Issue を確認し、要件を把握する
2. `feature_<Issue番号>` ブランチを作成する
3. 実装・テストを行う
4. PR を作成し、レビューを受ける
5. `main` にマージする

### コミット粒度

- 1 コミット = 1 つの論理的な変更単位
- 動作する状態でコミットする（テストが通る状態）
- WIP コミットは避ける
