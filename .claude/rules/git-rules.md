# Git ルール

## コミットメッセージ

[Conventional Commits](https://www.conventionalcommits.org/) に準拠する。

### プレフィックス

| プレフィックス | 用途 |
|---------------|------|
| `feat:` | 新機能 |
| `fix:` | バグ修正 |
| `chore:` | 雑務（依存関係更新、設定変更等） |
| `docs:` | ドキュメントのみの変更 |
| `refactor:` | 機能変更を伴わないコード整理 |
| `test:` | テストの追加・修正 |
| `style:` | フォーマット変更（コードの意味に影響しない） |
| `perf:` | パフォーマンス改善 |
| `ci:` | CI/CD 設定の変更 |

### 形式

```
<prefix>: <簡潔な変更内容（日本語可）>

<本文（任意）>

Refs: #<Issue番号>
```

- 件名は 72 文字以内
- フッターに `Refs: #<N>` で関連 Issue を紐付ける

## Pull Request

- **タイトル**: 日本語、形式 `[#<Issue番号>] <変更内容>`
  - 例: `[#48] Claude Code 設定ファイルの配置`
- **本文**: `/create-pr` スキルのテンプレートに従う

## ブランチ保護

- `main` ブランチへの直接プッシュは禁止
- `main` への force push は絶対に行わない
